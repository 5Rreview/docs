# 🏗️ Platform Architecture

High-level technical architecture of the 5★★★★★ platform.

## Overview

5R★ uses a multi-layer architecture: edge compute for speed, a modular monolith for business logic, and a multi-provider AI layer for intelligent analysis — all designed around the principle that trust and business features must remain strictly separated.

## ⚙️ Modular Backend

The backend uses a modular architecture where features are isolated into independent modules. Each module has its own models, controllers, services, views, routes, and migrations.

Key modules include:
- **Review** — 5R★ Reviews submission, display, and management
- **Subject** — companies and services being reviewed
- **AI Engine** — 5R★ AI Risk Flags, sentiment analysis, quality scoring, fake detection
- **Trust** — 5R★ Trust Score calculation, reputation weights, confidence intervals

## 🔒 Trust Layer Separation

The architecture enforces strict separation between trust and business functionality:

```
🛡️ Trust Layer (read-only to business)     💼 Business Layer (no trust influence)
├── 5R★ Trust Score Calculator             ├── Analytics Dashboard
├── 5R★ AI Risk Flags Pipeline             ├── Response Management Tools
├── Review Quality Scorer                  ├── CRM Features
├── Reputation Weight System               ├── Brand Profile Editor
└── Search Ranking                         └── Data Export / API
```

No data generated by business features flows into trust calculations. See [Separation Architecture](https://github.com/5Rreview/open-trust-methodology/blob/main/docs/separation-architecture.md).
