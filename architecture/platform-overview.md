# 🏗️ Platform Architecture

High-level technical architecture of the 5⭐⭐⭐⭐⭐ platform.

## Overview

5R★ uses a multi-layer architecture: edge compute for speed, a modular monolith for business logic, and a multi-provider AI layer for intelligent analysis — all designed around the principle that trust and business features must remain strictly separated.

## ⚙️ Modular Backend

The backend uses a modular architecture where features are isolated into independent modules. Each module has its own models, controllers, services, views, routes, and migrations.

Key modules include:
- **Review** — 5R★ Reviews submission, display, and management
- **Subject** — companies and services being reviewed
- **AI Engine** — 5R★ AI Risk Flags, sentiment analysis, quality scoring, fake detection
- **Trust** — 5R★ Trust Score calculation, reputation weights, confidence intervals

## 🔒 Trust Layer Separation

The architecture enforces strict separation between trust and business functionality. No data generated by business features flows into trust calculations.

### 🛡️ Trust Layer (read-only to business)

| Component | What it does |
|-----------|-------------|
| **5R★ Trust Score** | 4-pillar composite score (0–100): review-based, domain-based, profile-based, activity-based. Grades: Excellent → Critical |
| **5R★ AI Risk Flags** | 18 domain-level fraud signal types (registration, infrastructure, security, content, activity) + two-level review fake detection (L1 rules → L2 AI) |
| **5R★ Trust Signals** | Raw domain intelligence: WHOIS, SSL, DNS, Safe Browsing, IP reputation, tech stack — each with independent refresh cycles |
| **5R★ Reviews** | Quality scoring (5 metrics → A–F grade), sentiment analysis, review type weighting (community / verified / invited / editorial / seed / aggregated) |
| **5R★ Business Events** | Chronological timeline aggregated from 5 sources: news, badges, fraud signals, rating changes, company verification |

### 💼 Business Layer (no trust influence)

| Feature | Description |
|---------|-------------|
| Analytics Dashboard | Sentiment trends, competitor benchmarks |
| Response Management | Queue management, templates, team assignment |
| CRM Integration | Customer tracking, issue management |
| Brand Profile Editor | Media gallery, featured content, logo |
| Data Export / API | Bulk export, programmatic access |

### 🔄 How They Connect

```
5R★ Trust Signals (raw data)
  ├── feeds → 5R★ Trust Score (pillar calculation)
  ├── feeds → 5R★ AI Risk Flags (fraud detection)
  └── feeds → 5R★ Business Events (fraud signal events)

5R★ Reviews (user content)
  ├── feeds → 5R★ Trust Score (review-based pillar)
  ├── feeds → 5R★ AI Risk Flags (fake detection)
  └── feeds → 5R★ Business Events (rating change events)

5R★ AI Risk Flags
  ├── feeds → 5R★ Trust Score (lowers score if fraud detected)
  └── feeds → 5R★ Business Events (fraud signal timeline)

5R★ Business Events
  └── display only (no feedback into scoring)
```

See [Separation Architecture](https://github.com/5Rreview/open-trust-methodology/blob/main/docs/separation-architecture.md) for implementation details.
